I Built an AI Agent That Makes My Videos

Prabha 12 min read

The pitch: One Mac Mini M2. Zero SaaS subscriptions. An agent stack that 99% of creators pay $200+/month to approximate. This is how I actually run it — what works, what's broken, and why I retired my own Kanban system.

"The goal isn't to automate everything. The goal is to automate the boring parts so I can focus on what actually matters: building shit and telling honest stories about it."

What This Actually Is

I'm a Lead Product Designer at SuperOps. I also build iOS apps on the side (WhatPay, Time Left, a few others). I document my builds publicly. That documentation used to take 8+ hours a week.

Now it takes 30 minutes of review + approval.

The stack:

  • Hermes — My agent orchestration layer (Claude Code, cron jobs, skills)
  • Todoist — Single source of truth for all tasks
  • Honcho — Persistent memory across agent sessions
  • ComfyUI — Local image generation (no Midjourney subscription)
  • Hyperframes — Motion/animated content for YouTube intros
  • GitHub Pages — thisisprabha.com (everything public lives here)

What Worked

1. The Daily 6 PM Processor

Every evening at 6 PM, a cron job wakes up and does three things:

  • Scans my raw idea files ( scattered notes, screenshots, voice memos)
  • Extracts structured tasks, links, and references
  • Creates Todoist tasks with proper labels and priorities

I wake up to a clean, prioritized list. No more digging through 17 notes apps.

The magic The 6 PM job doesn't just dump tasks. It triages. "Urgent + Professional" vs "Personal + Backlog". It learned my patterns. I rarely have to re-priority anymore.

2. Skills as Disposable Expertise

I used to Google the same shit every 3 weeks. "How do I batch convert these images again?"

Now I convert that knowledge into a skill file. Each skill has:

  • Trigger conditions (when to use it)
  • Exact commands (copy-paste ready)
  • Pitfalls I've already hit

Current skill count: 20+. Some I wrote. Some I downloaded. Each one is reusable expertise.

3. Public-by-Default Documentation

Everything goes to GitHub Pages. Research briefs? HTML. Design explorations? HTML. This devlog? HTML.

Benefits I didn't expect:

  • Forces clarity — if I can't explain it publicly, I don't understand it
  • SEO happens automatically
  • People actually find and reference my work
  • No "where did I put that doc?" — it's always at thisisprabha.com/[thing]

What Broke

1. The Kanban Experiment (RIP)

I built a full Kanban automation with boards, issue tracking, auto-assignment. It was beautiful. It lasted 3 weeks.

The problem: It was a system around my work, not in my work. I had to context switch to use it. Todoist is already there. Already connected to everything.

Lesson: Automate within existing workflows. Don't invent new workflows just for automation.

2. Google OAuth (Perpetually Broken)

I've set up Google Workspace integration 4 times. It works for 2 weeks, then token expires, then I'm back to manual exports.

Current workaround: Python scripts that write directly to Docs via API. Less elegant. More reliable.

Status: Mitigated Using ~/.hermes/scripts/google_doc_from_text.py for all deliverables. Bypasses the OAuth dance entirely.

3. Context Compression Cost (Surprise)

Hermes compresses long conversations to stay under token limits. I had this routed through a paid API. Forgot about it. $47 bill one month.

Fix: Local compression via Kimi K2.6. Slightly slower. Infinitely cheaper.
