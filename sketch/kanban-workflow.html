What is Kanban here? ?

In the Hermes Agent world, Kanban is not a todo list. It is a multi-agent work queue where tasks move through real states — ready, todo, blocked, done — and different worker roles (orchestrator, researcher, analyst, writer) claim what they can handle.

Think of it as a shared whiteboard where agents pin jobs, grab the next sticky note, and move it to "done" without bothering me for every micro-decision.

Tasks move through four states on the board:

ready todo blocked done

Ready = queued, waiting for a worker to pick up.
Todo = actively being worked on.
Blocked = waiting on external input, approval, or a dependency.
Done = finished and shipped (or archived).

Not Todoist. Not Cron. Not Chat. ≠

Todoist

Operational layer for me. My human task list — reminders, habits, errands. I check boxes. Agents do not touch it.

Kanban

Multi-agent work queue. Autonomous pipelines — research, synthesis, drafting — moving through states without my hand on every card.

Cron

Scheduled triggers. "Run this at 6PM." Fires the orchestrator, which then populates the Kanban board with new tasks.

Manual Chat

Direct request. I ask, agent answers. No queue, no state machine. Best for one-offs, worst for pipelines.

The Kanban board is the middle layer between "I want this" and "it is shipped." Cron kicks it. Agents run it. I only step in when a rule says needs approval.

My 4 boards 📌

Tina · Auto-Run

Tina Board

Optimization, error-log fixes, setup patches — anything that improves the system without touching product or design.

Rule: auto-run everything. No approval loops.

Professional · Approval Gate

Professional Board

Research, competitive analysis, writing, strategy work. The bulk of my weekly pipeline lives here.

Rule: auto-research, needs approval for design/code.

App Work · Build Gate

App Work Board

Feature work, releases, builds, app store updates — anything that ships to users.

Rule: auto-plan, needs approval for builds/releases.

Personal · Manual

Personal Board

Side projects, experiments, learning queues. Mostly dormant unless I manually wake it.

Rule: manual/dormant. I move cards here.

Talking to the board ⌨

CLI commands I use to peek, prod, and push tasks:

hermes kanban list # show all boards + task states
hermes kanban show t_fcccbeae # inspect a specific task
hermes kanban move t_82ae71da done # manually advance (rare)
hermes kanban block t_26d47fa3 "waiting on API key" # mark blocked + reason

Worker lifecycle: Orchestrator spawns → researcher claims ready tasks → analyst picks up raw findings → writer drafts deliverable → done. If a task hits blocked, the orchestrator re-evaluates at the next cron tick (6PM, 7PM, or 3AM).

I rarely move cards manually. The agents handle state transitions. I only step in when a board rule says needs approval — then I get a ping, review, and say ship it or revise.

The board runs so I do not have to micro-manage every agent turn.
