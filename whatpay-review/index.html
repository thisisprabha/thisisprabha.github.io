WhatPay

Design & UX review of the WhatPay iOS app — a subscription & EMI tracker built for iPhone.

iPhone 16e Simulator Dark Mode 14 Screens iOS 26.2 Clean Capture
01
Onboarding

Welcome Screen

First-launch screen with a credit-card icon, headline copy, and a prominent "Get started" pill button. Escape hatches "Remind me later" and "Skip" sit at top corners.

  • Clean dark onboarding with clear hierarchy
  • Good use of iconography to set mental model
  • "Skip" placement is discoverable without being distracting
02
Onboarding

Country Selection

Two-column grid of country cards with currency symbols. India is pre-highlighted with a white fill and checkmark. "Continue" button disabled until selection.

  • Cards are large enough for thumb-friendly tapping
  • Missing flag icons shown as "?" placeholders — needs assets
  • Warning "cannot be changed later" is appropriately alarming
03
Onboarding

Country Selected State

India card now fully selected with white background, black text, and a checkmark. Other cards remain dark gray. "Continue" becomes active.

  • Strong visual contrast for selected vs unselected
  • One-tap selection — no confirmation step needed
  • Grid scrolls smoothly for additional countries
04
Onboarding

Quick Add Services

Grid of popular subscription services (Netflix, YouTube, Disney+, HBO Max, Spotify, Apple Music, etc.) with circular branded icons in 3-column layout.

  • Icon-driven discovery reduces cognitive load
  • 3-column grid maximizes screen real estate
  • Color coding helps brand recognition at a glance
05
Form

Service Price Entry

Pre-filled "Netflix" name with a price input field below. iCloud sync notice and "Add" action button at bottom.

  • Pre-filled name saves typing friction
  • Price keyboard should be numeric-first (verify)
  • Bottom button placement is thumb-reachable
06
Dashboard

Main Dashboard — First Card

Dashboard shows "My payments and emi's" header with AED total, monthly label, and a Netflix card showing ₹299. Bottom nav with list, "ADD NEW", and analytics icons.

  • Big bold total amount draws attention to spending
  • Card format is clean with clear cost visibility
  • Bottom action bar stays persistent across screens
07
Interaction

Expanded Card Detail

Tapping a card expands it to show full details: service name, cost, billing cycle toggle, and action buttons (Edit, Delete, etc.).

  • Expand/collapse pattern keeps dashboard clean
  • All actions visible without leaving the screen
  • Edit button is discoverable in expanded state
08
Form

Edit Subscription Form

Edit mode opens a form with name field, cost field, and cycle toggles. "Save" button at bottom. Form fields are clearly labeled.

  • Inline editing avoids modal navigation
  • Keyboard handling: verify field scrolls into view
  • Save/Cancel actions should be equally weighted
09
Dashboard

Dashboard After Update

Card collapses back to list view after save. Total recalculates if price changed. Smooth transition without jarring reloads.

  • Immediate visual feedback on save
  • Total updates in real-time
  • Animation polish: check for springy card collapse
10
Form

Add New Subscription

Empty form for manually adding a new subscription. Fields: name, cost, billing cycle, category picker. Clean dark form with subtle borders.

  • Consistent form pattern with edit screen
  • Category picker adds taxonomy for future filtering
  • Needs field validation messages (verify)
11
Analytics

Insights / Spending View

Scroll-down reveals insights section with category breakdown, upcoming payments, and spending trends. Graph visualizations in dark theme.

  • Scroll-reveal pattern keeps primary actions accessible
  • Charts need sufficient contrast in dark mode
  • Upcoming payments = high-value reminder feature
12
Navigation

Dashboard Scroll State

Main dashboard scrolled to show additional cards below the fold. Multiple subscription cards visible with costs and icons.

  • Scrolling is smooth with no jank
  • Cards maintain consistent spacing and alignment
  • Bottom nav bar stays fixed during scroll
